# 결제 모듈 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
패션 쇼핑몰 경제 모듈

### 사용하게 될 기술: 
React

### 문서 작성자: 
인턴IN메타장윤영

## 📝 고객 요구사항 정리
여러 개의 카드를 등록할 수 있게 설계
카드 등록 과정에서 민감한 정보가 바로 노출되지 않도록 보안
이 결제 모듈의 경우, 상품 목록에서 담기 버튼 바로 옆에 '결제' 버튼을 추가하여 동작할 수 있게 설계

## 📋 기능 목록
- 카드 번호 input
  - `<CardNumberInput />`
  - 숫자가 아닌 것은 입력 불가능
- 카드 만료일 input
  - `<ValidityInput />`
  - 월/연도 형식으로 작성
  - 숫자가 아닌 것은 입력 불가능
- 카드 소유자 이름 input
  - `<NameInput />`
  - 30자 이내로 작성
  - 영문이 아닌 것은 입력 불가능
- 보안코드 input
  - `<SecretInput />`
  - 보안 형식으로 표현
  - 민감정보 숨김처리
- 카드 비밀번호 input
  - `<PasswordInput />`
  - 보안 형식으로 표현
  - 민감정보 숨김처리
- 카드 목록 input
  - `<CardListInput />`
  - 이미 등록한 카드가 있는 경우 등록한 카드는 위쪽에 배치하고 신규로 입력하려면 아래쪽에 위치
